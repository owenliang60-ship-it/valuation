# Position Sizing Framework

> 基于 OPRMS 评级系统的仓位管理框架
> 核心公式: `最终仓位 = 总资产 x DNA上限 x Timing系数`

---

## 1. Core Formula

```
Final Position ($) = Total Capital x DNA_cap x Timing_coefficient
Final Position (%) = DNA_cap x Timing_coefficient
```

**示例**: 总资产 $5,000,000, DNA = A (15%), Timing = A (0.9)
- 最终仓位 = $5,000,000 x 0.15 x 0.9 = **$675,000 (13.5%)**

---

## 2. DNA Cap Table

| DNA Rating | 名称 | Max Position % | 特征 | 持有期 |
|-----------|------|---------------|------|-------|
| **S** | 圣杯 Holy Grail | 20-25% | 改变人类进程的超级核心资产，护城河极深，ROIC 持续 20%+，TAM 巨大 | 永久 |
| **A** | 猛将 General | 15% | 强周期龙头，细分赛道霸主，ROIC 15%+，管理层优秀 | 3-10 年 |
| **B** | 黑马 Dark Horse | 7% | 强叙事驱动，赔率高但不确定性高，可能是早期颠覆者 | 6月-3年 |
| **C** | 跟班 Follower | 2% | 补涨逻辑，流动性溢出，基本面一般 | 1-6 月 |

### DNA 评级变更条件
- **升级**: 需显著正面新证据（护城河拓宽、TAM 扩大、管理层正面变化）
- **降级**: 出现结构性负面变化（竞争加剧、护城河收窄、管理层失信）
- **禁止**: 因短期价格波动改变 DNA 评级
- **记录**: 所有变更记入 Knowledge Base rating changelog

## 4. Evidence Threshold (证据门槛)

仓位大小与证据充分程度挂钩：

| 阶段 | 允许仓位 | 证据要求 |
|------|---------|---------|
| 初始假设 | 1/3 目标仓位 | 初步论文 + 至少 1 个主要信源 |
| 论文确认 | 2/3 目标仓位 | 2+ 主要信源 + Variant View 成型 |
| 完整证据 | 满仓 | 3+ 主要信源 + 8-10 总信源 + 完整备忘录 (>7.0分) |
| 超额配置 (S-Timing) | >目标仓位 | 上述全部 + 历史性时机确认 |

---

## 5. Scaling Protocol

### 建仓
```
Step 1: 1/3 仓位 — 初步论文成立，基本面扫描通过
Step 2: +1/3 仓位 — 论文得到新证据确认，Kill Conditions 定义完成
Step 3: +1/3 仓位 — 完整备忘录完成，3+ 主要信源，评分 > 7.0
```
- 每次加仓间隔 >= 1 周（避免 FOMO）
- 每次加仓需在 Trading Desk 交易日志中记录理由

### 减仓
```
Kill Condition 触发 → 立即全部退出
-15% 回撤 → 强制审查，可能减至 1/2
-25% 回撤 → 减至 1/3 或全部退出
论文实现 → 分批止盈 (1/3 → 1/3 → 1/3)
Timing 降级 → 按新系数调整仓位
```

---

## 6. Portfolio-Level Constraints

| 约束 | 限制 |
|------|------|
| 总股票仓位 | <= 100% (不加杠杆) |
| 现金储备 | >= 5% |
| S 级满仓数量 | <= 2 个同时 |
| 单行业合计 | <= 40% |
| 前 3 大持仓合计 | <= 50% |
| 做空总暴露 | <= 15% |
